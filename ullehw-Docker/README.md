Основная идея
Необходимо создать два образа и написать docker-compose файл, с помощью которых будут развернуты 3 связанных сервиса.

Подробное описание
С помощью docker compose и Dockerfile необходимо создать 3 сервиса. Все сервисы должны находиться в одной docker-сети типа user-defined bridge (не дефолтный!). В сервисе-1 нужно использовать указанный готовый образ, все сервисы нужно называть так, как указано в графе Название сервиса.

Сервис-1. База данных
Образ - mariadb
Название сервиса - db

В БД будут храниться данные для веб-приложения Сервиса-3. Данные представляют собой два столбца:
name (строковый)
age   (целочисленный)

Сервис-2. Скрипт для заполнения БД

Образ - самодельный
Название сервиса - filler

Сервис-2 дожидается, пока Сервис-1 запущен и готов принимать подключения, затем к нему по сети и заполняет набором данных.

Примечания:
можете писать скрипт на любом языке, советуем Python или Bash
нужно написать Dockerfile, с помощью которого можно собрать образ для контейнера Сервиса-2
Требования:
Набор данных представлен CSV-файлом data.csv с двумя колонками (name, age) с заголовком
Сервис-2 стартует после старта Сервиса-1, когда Сервис-1 готов принимать запросы
Сервис-2 останавливается после заполнения БД
Сервис-2 выводит содержимое БД после ее заполнения в stdout (проверка корректности заполнения)

Сервис-3. Веб-сервер

Образ - самодельный
Название сервиса - web

Контейнер с веб-сервером, который может отдавать данные из Сервиса-1 по http-запросу.
